# Core Foundation Required Fields (Locked v1)

This file defines the minimum brand foundation data model that powers strategy, standards, activation plans, prompts, and exports.

Use this as the source of truth for:
- workbook field requirements
- tier-level visibility/editability
- PDF/download completeness checks

## Principle

If a field does not materially improve message quality, conversion clarity, or execution speed, it should not be required.

## Required Foundation Blocks

### 1) Positioning Core
- Brand/category statement
- Primary audience statement
- Core problem solved
- Unique value/mechanism
- Expected outcome

### 2) Elevator Pitch + Company Description
- Elevator pitch (1 sentence)
- Elevator pitch (30-second version)
- Elevator pitch (60-second version)
- Boilerplate short (50-75 words)
- Boilerplate medium (100-150 words)
- Boilerplate long (200-300 words)

### 3) ICP + Persona Core
- ICP hierarchy (primary, secondary, tertiary)
- Persona JTBD (by ICP)
- Frictions/pains (by ICP)
- Objections (by ICP)
- Buying triggers (by ICP)

### 4) Messaging Architecture
- Messaging pillars (3-5 max)
- Claim + proof point per pillar
- Audience-specific one-liners
- CTA by funnel stage (Awareness / Consideration / Decision)

### 5) Voice + Standards Language
- Voice attributes
- Do / Don't examples
- Lexicon (approved terms)
- Blocklist (forbidden terms/phrases)
- Reading level + writing style rules

### 6) Differentiation + Competitive Lens
- Key alternatives/competitors
- Where we win
- Counter-positioning statements
- Risk watchouts (simple SWOT-style risks/opportunities)

### 7) Offer + Conversion Narrative
- Core offers and for-whom mapping
- Offer outcome statement
- Lead magnet alignment (if included in tier)
- Primary conversion path (entry CTA -> next CTA)

### 8) Proof Library
- Metrics/stat snippets
- Testimonial snippets
- Credibility markers (awards, logos, certifications)

### 9) Governance Essentials
- Messaging owner
- Review cadence (monthly/quarterly)
- Version note / change log

## Downstream Dependency Map (What each block feeds)

- Positioning Core -> strategy tab narrative, homepage/LP messaging, one-page messaging.
- Elevator Pitch + Boilerplate -> website hero/about, PR/profile use, role-pack leadership.
- ICP + Persona Core -> messaging matrix filters, channel plans, nurture/social segmentation.
- Messaging Architecture -> activation drafts, prompt outputs, campaign briefs, sales narratives.
- Voice + Standards Language -> brand standards docs, voice checklist, design/marketing packs.
- Differentiation + Competitive Lens -> competitive brief, battle cards, objection handling.
- Offer + Conversion Narrative -> funnel CTA logic, lead magnet planning, nurture flow.
- Proof Library -> social proof insertions across web/email/sales/pitch materials.
- Governance Essentials -> workbook versioning, regeneration confidence, team handoff consistency.

## Keep vs. Cut Guidance

Keep:
- items directly reused in at least 2 delivery surfaces (activation + downloads, standards + website, etc.)
- concise fields with execution implications

Cut or demote:
- enterprise-only frameworks with no short-term action path
- duplicate narrative fields that restate the same concept
